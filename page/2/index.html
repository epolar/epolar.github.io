Junit 测试Android

发表于 2015-12-03   |   分类于 Android笔记

Junit(单元测试)，好吧，我看啊看我依然不明白它有什么作用。只是对于单一的方法或许可以一测吧。



每个JUnit测试都可以包含：测试前环境配置 –> 测试用例 –> 测试后资源重置。

在看Android之前的，就先看看Java的吧，Java的比较简单，使用的是JUnit 4做的测试，在使用Eclipse的时候可以自动添加JUnit 4环境。


Java的JUnit测试主要就是使用注解 @Test 注解一个方法表明该方法是一个测试用例，用 @After 注解来对 @Test 方法运行完进行后续处理， @Befor 注解对 @Test 方法运行前进行预先的配置处理，不需要西继承什么类了。Java在Eclipse中在运行的时候选择JUnit就可以进行JUnit测试了。在视图里可以找到”JUnit”来查看结果。

先创建一个被测试的类：


public class JunitTest {


public int add(int arg1, int arg2) {

return arg1 + arg2;

}


public int sub(int arg1, int arg2) {

return arg1 - arg2;

}


}



编写测试代码：


public class MyTestCase {


int arg1 = 10;

int arg2 = 3;

int r1 = 13;

int r2 = 7;

JunitTest jt = null;


@Before

/

每个测试用例被调用前调用

/

public void init() {

jt = new JunitTest();

}


@Test

/

测试用例

/

public void testAdd() {

assertTrue(“That is”, jt.add(arg1, arg2) == r1);

}


@Test

/

测试用例

/

public void testSub() {

assertTrue(“sub is”, jt.sub(arg1, arg2) == r2);

}


@After

/

每个测试用例被调用后调用

/

public void end() {

System.out.println(“Test finish”);

}


}



Android在SDK1.5后提供了JUnit测试，Android的 JUnit 测试需要继承 TestCase 类，并且需要在ManiFest.xml中进行配置，可以写一个类，内部包含一个 public static Test suite(){} 方法来管理测试用例类。Android的JUnit可以是没有界面的(JUnit隐藏掉界面)，不过像下面这样的例子，跟Java基本上就是一致的，这也是最简单的用例。Android与JUnit的结合核心或许是在 BaseTestRunner 类和 AndroidTestRunner 类(junit.runner包)。默认的安卓会有一个BaseTestRunner来帮助我们运行代码，我们也可以通过自己去实例化一个AndroidTestRunner和实现自己的TestListener来做测试。


在Android里，可以用 Dev Tools(AVD中的APP) –> Instrumentation –> instrumentation结点中的 label 来测试

Android的JUnit集成在内部，所以可以在API文档中查到相关的包、类等。Android中的JUnit包括如下的包：

测试的对象跟Java一样，先写一个被测试的类：


public class SampleCalcutor {


public int add(int arg1, int arg2) {

return arg1 + arg2;

}


public int sub(int arg1, int arg2) {

return arg1 - arg2;

}


}



测试类继承于 TestCase，其中的 setUp() 方法在测试样例被调用前调用，可用于环境配置， tearDown() 方法在测试样例调用结束后调用，可用于资源回收， 测试样例只需要是以 test 开头就可以被检查出来(相应的也就是说，test开头的方法都将被当成测试用例)：


public class MyJUnitTest extends TestCase {


int arg1 = 10;

int arg2 = 3;

int r1 = 13;

int r2 = 7;

SampleCalcutor sc = null;


@Override

/

每个测试样例调用前调用，可以用于环境配置

/

protected void setUp() throws Exception {

// TODO Auto-generated method stub

super.setUp();

sc = new SampleCalcutor();

System.out.println(“测试前”);

}


@Override

/

每个测试样例结束后调用，可以用于资源的回收

/

protected void tearDown() throws Exception {

// TODO Auto-generated method stub

super.tearDown();

System.out.println(“测试完”);

}


/

测试用例1

/

public void testAdd() {

int result = sc.add(arg1, arg2);

assertTrue(“add result is “, result == r1);

}


/

测试用例2

/

public void testSub() {

int result = sc.sub(arg1, arg2);

assertTrue(“subtract result is “, result == r2);

}


}



以上已经可以运行了，通过右键项目 – Run As –> Android JUnit Test， 不过这样如果有多个 TestCase ，就都会被运行出来，所以我们可以用一个拥有 public static Test suite(){} 的类来运行，使用 TestSuite 管理 TestSuite.addTestSuite() 添加需要测试的用例类：


public class TestAll {


/*

管理测试用例，直接在TestAll右键以Android JUnit方式运行

@return

/

public static Test suite() {

TestSuite suite = new TestSuite();

suite.addTestSuite(MyJUnitTest.class); // 测试用例 class

return suite;

}

}

Android 4.4 (SDK_INT >= 19) 以后，状态栏的透明

发表于 2015-10-28   |   分类于 Android笔记

4.4以后才支持的沉浸式ActionBar+StatusBar。

在Java代码中，可以通过

getWindow().setFlags(WindowManager.LayoutParams.FLAG_TRANSLUCENT_STATUS |

WindowManager.LayoutParams.FLAG_TRANSLUCENT_NAVIGATION,

WindowManager.LayoutParams.FLAG_TRANSLUCENT_NAVIGATION | WindowManager.LayoutParams.FLAG_TRANSLUCENT_STATUS);

进行设置，另外还有一种方法，效果和这个不一样，它是全屏的，状态栏透明且可以下滑：

mDecorView.setSystemUiVisibility(

View.SYSTEM_UI_FLAG_LAYOUT_STABLE

| View.SYSTEM_UI_FLAG_LAYOUT_HIDE_NAVIGATION

| View.SYSTEM_UI_FLAG_LAYOUT_FULLSCREEN

| View.SYSTEM_UI_FLAG_HIDE_NAVIGATION // hide nav bar

| View.SYSTEM_UI_FLAG_FULLSCREEN // hide status bar

| View.SYSTEM_UI_FLAG_IMMERSIVE);

在 xml style 中的设置方式：

<item name=“android:windowTranslucentStatus”>true</item>

<item name=“android:windowTranslucentNavigation”>true</item>

不过在 xml 中，不知道出于什么原因，我继承 AppTheme 后，给一个Activity(不是Application) 设置了这个style，结果无效。

在5.0中，可以设置标题栏颜色：

<item name=“android:colorPrimary”>?attr/bg_title_color</item> // ActionBar

<item name=“android:colorPrimaryDark”>?attr/bg_title_color</item> // 状态栏

<item name=“android:navigationBarColor”>?attr/bg_title_color</item> // 导航栏

这些基于原生的设置，可能不是很方便(简单的说呢，就是使用 translucentStatus 的时候，布局会跑到状态栏背后)，可能需要添加一个View或设置Margin/Padding来让它不跑上去，这就需要获取ActionBar的高度及StausBar的高度了：

各系统控件的高度代码获取(代码取自 SystemBarTint 的 SystemBarConfig)：

private static final String STATUS_BAR_HEIGHT_RES_NAME = “status_bar_height”; // 状态栏高度

private static final String NAV_BAR_HEIGHT_RES_NAME = “navigation_bar_height”; // 导航栏高度

private static final String NAV_BAR_HEIGHT_LANDSCAPE_RES_NAME = “navigation_bar_height_landscape”;

private static final String NAV_BAR_WIDTH_RES_NAME = “navigation_bar_width”;

private int getInternalDimensionSize(Resources res, String key) {

int result = 0;

int resourceId = res.getIdentifier(key, “dimen”, “android”);

if (resourceId > 0) {

result = res.getDimensionPixelSize(resourceId);

}

return result;

}

或通过开源项目 SystemBarTint 可以达到一个相对不错的效果，SystemBarTint 设置沉浸式以后，添加代码：

mSystemBarTintManager = new SystemBarTintManager(this);

mSystemBarTintManager.setStatusBarTintEnabled(true);

mSystemBarTintManager.setNavigationBarTintEnabled(true);

// 获取到颜色

TypedArray _ta = getTheme().obtainStyledAttributes(YogaUtil.getInstance().getThemeResId(), new int[]{R.attr.bg_title_color});

mSystemBarTintManager.setStatusBarTintColor(_ta.getColor(0, getTitleColor()));

_ta.recycle();

获取app在特定dpi下的图片(指定dpi的activity icon)

发表于 2015-10-28   |   分类于 Android笔记

获取 res 下的图片，最重要的是

Package.getResourcesForActivity(component).getDrawableForDensity(resId, Display.Metrics.XXHIGH);

这个方法，即通过DrawableForDensity去获取，这个方法的加入是API 15，但是在14的时候已经可以调用，更早的没有测试。

以下是获取一个Activity的Icon的代码：

/**

* 通过包名和Activity名获取Activity的icon,如果得不到activity的icon则获取包的

* @param packageName

* @param activityName

* @return

*/

@SuppressLint(“NewApi”)

public Drawable getActivityIcon(String packageName, String activityName) {

ComponentName component = new ComponentName(packageName, activityName);

PackageManager pm = mContext.getPackageManager();

try {

int iconId = pm.getActivityInfo(component, 0).icon;

if (iconId == 0) {

iconId = pm.getApplicationInfo(packageName, 0).icon;

}

if (iconId > 0 && Build.VERSION.SDK_INT >= 14) {

// Method method = Resource.class.getMethod(“getDrawableForDensity”,

int.class, int.class);

// return (Drawable) method.invoke(pm.getResourcesForActivity(component),

iconId, DisplayMetrics.DENSITY_XXHIGH);

Drawable d = null;

int displayMetrics[] = { 640, DisplayMetrics.DENSITY_XXHIGH,

DisplayMetrics.DENSITY_XHIGH,

DisplayMetrics.DENSITY_HIGH };

for (int dm : displayMetrics) {

d = pm.getResourcesForActivity(component).getDrawableForDensity(iconId, dm);

if (d != null) {

return d;

}

}

return pm.getActivityIcon(component);

} else {

return pm.getActivityIcon(component);

}

} catch (Exception e) {

try {

return pm.getApplicationIcon(packageName);

} catch (NameNotFoundException e1) {

return null;

}

}

}

这个方法得到尽可能大(极限是 xxxhdpi)的ActivityIcon，当得不到ActivityIcon就拿PackageIcon



 

 

ImageSpan 可点击、图片与设置不一样大

发表于 2015-10-28   |   分类于 Android笔记

ImageSpan可以在TextView中嵌入图片。又或者说，android.text.style.* 里面的东西可以放在 SpannableString 里再设置到 TextView 里可以使 TextView 显示出不同的效果（ Html.fromHtml() 就是返回一个SpannableString或Spannabed）。在设置 ImageSpan 的时候，出现了一个问题是设置进行的 Bitmap 画出来的大小并不是原来 Bitmap 的大小，经过源码查看，发现是 ImageSpan 当中通过 new BitmapDrawable(context.getResoutces(), b) 或 new BitmapDrawable(b) 生成 Drawable 导致的。

根据 api ，new BitmapDrawable的时候不传入 Context ，则是不处理密度生成Drawable，这个时候就会出现设置进去的 Bitmap 跟画出来的图片大小不一致，达不到我想要的效果。

TextView设置SpannableString，也是可以做点击的，对于 Span 的监听，主要是在 TextView.setMovementMethod() 方法当中传入的方法中的问题，原本我们可以通过 TextView.setMovementMethod(new LinkMovementMethod()) 使 ClickableSpan(UrlSpan) 生效，但是ImageSpan本身并没有实现ClickableSpan，而在 LinkMovementMethod 中，处理的只有ClickableSpan，所以我们重写卫LinkMovementMethod，使之处理 ImageSpan， 同时需要重写 ImageSpan 使之拥有 onClick (或其他方法，只要在 LinkMovementMethod 的 onTouchEvent 中调用即可)。

参考资料： http://www.cnblogs.com/luction/p/3645210.html

 

 

只在主界面应用上显示的悬浮窗

发表于 2015-10-28   |   分类于 Android笔记

悬浮窗是把控件添加到WindowManager并且设置其 WindowManager.Layoutparams设置为包含控件大小、类型为ALERT等可弹出的等。

悬浮窗用处很多，其中包括：ListView/GridView的拖动、Toast等

/**

* 在 manifest.xml 中应该配置权限 android.permission.SYSTEM_ALERT_WINDOW

* 为了只在主界面应用才显示，需要获取任务栈，

* 需要权限 android.permission.GET_TASKS

* 把悬浮窗布局/控件添加到 WindowManager 上，

* 并且设置 WindowManager.LayoutParams 即可，

* 通过监听触摸事件，并且更新 WindowManager.LayoutParams 的x,y就可以更新位置

* @author: 思落羽

* @date: 2014-2-12

* @Description:

*/

public class SuspendButtonService extends Service {

public static final String ACTION_OPERATE = “action_operate”;

public static final short OPERATE_SHOW = 1;

public static final short OPERATE_HIDE = 2;

protected boolean added = false;

protected boolean show = false;

protected WindowManager.LayoutParams wmParams;

/**

* 检查当前是否为主界面应用

*/

protected Handler hCheckMain;

/**

* 主界面包名

*/

protected List<String> homeNames;

protected Button btn;

@Override

public IBinder onBind(Intent intent) {

return null;

}

@Override

public void onCreate() {

super.onCreate();

init();

}

@Override

public void onStart(Intent intent, int startId) {

if (intent != null) {

/ 获取操作来确定当前应该显示或是取消显示 /

short operate = intent.getShortExtra(ACTION_OPERATE, (short)0);

switch(operate) {

case OPERATE_SHOW:

added = true;

show();

break;

case OPERATE_HIDE:

added = false;

hide();

break;

}

}

}

/**

* 初始化共用的对象

*/

protected void init() {

/* 2003是 WindowManager.LayoutParams.TYPE_SySTEM_ALERT 的值

初始化 WindowManager.LayoutParams /

wmParams = new WindowManager.LayoutParams(WindowManager.LayoutParams.WRAP_CONTENT,

WindowManager.LayoutParams.WRAP_CONTENT, 2003,

WindowManager.LayoutParams.FLAG_NOT_FOCUSABLE,

PixelFormat.RGBA_8888); // 类型要为透明才不会把背景也覆盖掉

/ 获取悬浮窗控件 /

btn = (Button) LayoutInflater.from(this)

.inflate(R.layout.demo_suspend_button, null);

btn.setOnTouchListener(new TouchBtn());

btn.setOnClickListener(new OnClickListener() {

@Override

public void onClick(View v) {

Toast.makeText(SuspendButtonService.this, “点击了悬浮窗”, Toast.LENGTH_SHORT).show();

}

});

}

/**

* 获取到桌面应用的应用名称

* @return

*/

protected List<String> getHomes() {

List<String> names = new LinkedList<String>();

PackageManager packageManager = getPackageManager();

/* 找出action为 Intent.ACTION_MAIN，

Category为Intent.CATEGORY_HOME的包的包名，这个包就是主界面的包 /

Intent intent = new Intent(Intent.ACTION_MAIN);

intent.addCategory(Intent.CATEGORY_HOME);

/ 查找intent可以唤醒的Activity的信息 /

List<ResolveInfo> resolveInfos = packageManager

.queryIntentActivities(intent, PackageManager.MATCH_DEFAULT_ONLY);

for (ResolveInfo resolveInfo : resolveInfos) {

names.add(resolveInfo.activityInfo.packageName);

}

return names;

}

/**

* 把悬浮窗添加到 WindowManager上

*/

protected void show() {

if (!show) {

show = true;

/ 获取 WindowManager /

WindowManager wm = (WindowManager) getSystemService(Service.WINDOW_SERVICE);

wm.addView(btn, wmParams);

checkMain();

}

}

/**

* 把悬浮穿从 WindowManager 上移除

*/

protected void hide() {

if (show) {

show = false;

/ 获取 WindowManager /

WindowManager wm = (WindowManager) getSystemService(Service.WINDOW_SERVICE);

wm.removeView(btn);

}

}

/**

* 检查当前界面是否在主界面，悬浮穿控件为只在主界面显示

*/

protected void checkMain() {

/ hCheckMain 为空就新建 /

if (hCheckMain == null) {

homeNames = getHomes();

hCheckMain = new Handler(new Handler.Callback() {

@Override

public boolean handleMessage(Message msg) {

if (isHome()) {

if (!show)

show();

} else {

if (show)

hide();

}

if (added) {

/ 1秒后扫描当前是否需要显示 /

hCheckMain.sendEmptyMessageDelayed(0, 1000);

}

return true;

}

});

}

ksoap2 连接 webservice

ksoap2 较为轻型，适合Android使用，Android没有内置WebService工具，WebService传回数据是标准的XML，可以用SoapObject直接得到结点值


ksoap2 code.google.com地址：https://code.google.com/p/ksoap2-android/

下载的ksoap2-android-assembly-x.x.x-jar-with-dependencies.jar文件放到libs中就可以使用了。

ksoap2主要使用对象： SoaoObject(连接参数描述)、 SoapSerializationEnvelope(连接描述)、HttpTransportSE(发起连接)、SoapFault/SoapFault12(错误描述，可以打印错误栈)





private static final String NAME_SPACE = “http://WebXml.com.cn/“;

private static final String URL = “http://www.webxml.com.cn/WebServices/ChinaZipSearchWebService.asmx”;

private static final String METHOD_NAME = “getSupportProvince”;

private static final String SOAP_ACTION = NAME_SPACE + METHOD_NAME; // 通常为NAME_SPACE + METHOD_NAME，为null会连接不上



public void conn() {



/ 生成SoapObject对象，指明要连接的对象 第一个参数为命名空间，第二个参数为方法 /
SoapObject so = new SoapObject(NAME_SPACE, METHOD_NAME);
/ 方法携带的参数 /
// so.addProperty(“tns:getWeatherbyCityNameSoapIn”, “福建”);
/ SOAP信息描述，传入的参数是WebService soap版本 /
SoapSerializationEnvelope sse = new SoapSerializationEnvelope(SoapSerializationEnvelope.VER11);
/ 设置输入接收对象 /
sse.bodyOut = so;
sse.setOutputSoapObject(so);
sse.dotNet = true; // 这行功能未知，注释掉同样可以取得连接，wsdl的必须为true
/ 创建HttpTransportSE，这个对象描述了WebService的URL，由它来进行连接 /
HttpTransportSE ht = new HttpTransportSE(URL);
try {
/ 开始连接 /
ht.call(SOAP_ACTION, sse);
/ 获取连接结果，造成错误经常是代码里的问题，如果测试通过应该不管是否成功都返回SoapObject */
if (sse.bodyIn instanceof SoapObject) {
SoapObject rso = (SoapObject) sse.bodyIn;
SoapObject result = (SoapObject) rso.getProperty(METHOD_NAME + “Result”);
for (int i = 0; i < result.getPropertyCount(); i ++) {
System.out.println(result.getProperty(i));
tv.append(result.getProperty(i).toString() + “\n”);
}
} else {
SoapFault sf = (SoapFault) sse.getResponse();
sf.printStackTrace();
}
} catch (HttpResponseException e) {
// TODO Auto-generated catch block
e.printStackTrace();
} catch (IOException e) {
// TODO Auto-generated catch block
e.printStackTrace();
} catch (XmlPullParserException e) {
// TODO Auto-generated catch block
e.printStackTrace();
}


}

NotificationManager和Notification

发表于 2015-10-27   |   分类于 Android笔记

Notification 是下拉列表与状态栏信息显示控件，最常见的案例是短信 —– 收到短信的时候，状态栏会显示通知提示，LED灯亮起，提示音响起，手机震动，下拉栏里新增该信息的通知，点击该通知将跳转到信息Activity中

// 得到 NotificationManager 对象，这个对象的获取通过 Context 的 getSystemSerivce 可以得到

NotificationManager nm = (NotificationManager) this.getSystemService(Context._NOTIFICATIONSERVICE);

// 创建一个 Notification 对象， 这里传入的三个值是显示在状态栏上的，其中第二项将会在刚显示的时候出现

Notification notification = new Notification(android.R.drawable._stat_notifychat, “Hello World”, System.currentTimeMillis());

// 点击了该 Notification(通知) 打开的窗口

Intent notificationIntent = new Intent(this, ListActivity.class);

// Notification 的Intent

PendingIntent contentIntent = PendingIntent. getActivity(this, 0, notificationIntent, 0);

// 调用系统 Notification 的话，在下拉列表里显示的信息(图标已经在上面设置过)

notification.setLatestEventInfo( this, “我是Title” , “这是一条通知”, contentIntent);

// 显示 Notification

nm.notify(0, notification);

PendingIntent 的 requestCode 如果设置成同一个值，那么就会根据 flag 进行相关操作，两个 PendingIntent被认为是同一个

自定义 Notification 样式的核心是设置contentView为一个RemoteView
